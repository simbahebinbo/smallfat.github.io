Keypoints of "The hacker guide to python"

发表于 2016-07-12

The code style for writing python code

  • use 4 spaces per indentation level
  • limit all lines a max of 79 chars
  • seperate top-level function and class-definitions with 2 blank lines
  • encoding file using ASCII or utf-8
  • one module import per import statement and per line, at the top of the file, after comments and docstrings, grouped first by standard, then third party, and finally local library imports.

The zen of Python, by Tim Peters

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
Special cases aren’t special enough to break the rules.
Although practicality beats purity.
Errors should never pass silently.

The internals of the ‘import’ subsystem

  • The list of modules currently imported is available through the sys.modules variable
  • builtin modules are listed in sys.builtin_module_names.
  • where to look for modules to load： sys.path.append(‘xxx’) or PYTHONPATH env variable.

The decorator

  • what: a function that takes another function as an argument and replace it with a new, modified function.

The generator

  • what: an object that returns a value on each call of its next() method until it raises StopIteration. It offers an easy way to create objects that implement the iterator protocol.
  • how: write a normal python function that contain a yield statement.

    i.e.: xrange(integer) next() yield

The list comprehension

  • what： allow to define a list’s contents inline with its declaration.

    i.e.: x=[i for i in (1,2,3)]

The functional functions funtioning

  • what: builtin functions

    i.e.: map(function, iterator) filter(function, iterator) enumerate(iterator) sorted(iterator) any(iterable)
